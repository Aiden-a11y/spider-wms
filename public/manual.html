Rate Version2026-02-17

01Overview

The Spider WMS Billing System allows your team to generate accurate monthly invoices for 3PL customers by automatically pulling order data from the WMS API and organizing charges into six billing categories.

📥

Auto-Fill from WMS

Click Load WMS Data to auto-populate inbound, B2B, B2C, and labor quantities directly from the WMS API for the billing period.

✏️

Adjust & Override

All auto-filled quantities can be manually overridden per order in the WMS Source Data table. Edits are saved with the invoice.

📊

Export to Excel

One-click export generates a fully-formatted, formula-linked Excel workbook with per-customer tabs, a summary sheet, rate table, and raw data sheets.

Billing Categories

📦 Inbound Handling
Cartons, pallets, containers (20'/40'/40HC, palletized or floor-loaded), additional labor
🗄️ Storage
Bin, Shelf, Carton, Pallet Short/Regular/Tall, Open Floor — billed as avg of 15th-day and last-day snapshot
🚚 Fulfillment B2B
Order processing, pick (piece/carton/pallet), packing, palletizing, labels, inserts
🛍️ Fulfillment B2C
Order processing, pick per piece, fragile handling, labels, inserts
🔄 Return Management
Return processing fee, restocking, disposal (cost + 10%)
👷 Warehouse Labor
Regular labor hours, OT weekday, OT weekend/holiday

04Creating an Invoice

Click + New Invoice in the top-right corner. A form slides in from the right where you select the customer(s) and billing period.

Single Invoice

  1. Select customer from the dropdown (pulls from WMS customer list)
  2. Select year & month for the billing period
  3. Click Create Invoice
If a Rate Master has been configured for this customer, custom rates are automatically applied when the invoice is created.

Combined Invoice (Multi-Customer)

  1. Click Select Multiple to enter multi-select mode
  2. Check each customer to include
  3. Select year & month (same period for all)
  4. Click Create Combined Invoice
All customers in a combined invoice share the same OM Subsidy, Office Sublease, and Summary panels. Each customer has its own WMS source data and billing line items.

06WMS Source Data Panel

The WMS Source Data panel appears at the top of each customer's invoice. It shows the raw order data fetched from the WMS API and lets you override individual quantities per order before they're summarized into billing line items.

Auto-persistence: WMS source data is saved together with the invoice. When you reopen an invoice, the source table is automatically restored — no re-fetch needed. Click Load WMS Data to re-fetch and overwrite with fresh data.

Loading Storage Data

  1. Set snapshot dates
    기준일 1 = 15th of the billing month · 기준일 2 = last day of the billing month
  2. Load from WMS History
    Click the purple button to auto-fetch location occupancy counts from the WMS History API for both dates. The system counts distinct locations per type, filtered to this customer.
  3. Review the counts
    The left panel shows 15th-day counts, right panel shows last-day counts. Location types include: Bin, Shelf, Carton, Pallet Short, Pallet Regular, Pallet Tall, Open Floor. Re-prefix types (Re Bin, Re Shelf, etc.) are automatically counted under the same category.
  4. Apply to Invoice
    Click Apply to Invoice to push the average values into the billing line items. This happens automatically when WMS History loads.
  5. Alternative: Upload Excel
    If WMS History is unavailable, drag-and-drop or select an Excel export from WMS containing Location, occupancyInfo (or Location Type), and Customer columns.
Formula: Avg Billed Qty = (15th Day Qty + Last Day Qty) ÷ 2. Each storage type row in the invoice uses this average as the billable quantity.

13Saving & Finalizing

💾 Save All (Draft)

Saves the current state of all invoices in the group to the database (Redis). The editor stays open. The last-saved timestamp appears in the header.

Saved data includes: all line items, WMS source data, per-order edits, OM Subsidy settings, Storage snapshots.

✓ Finalize All

Marks all invoices in the group as Final. Use this when the invoice is approved and ready to send to the customer.

Note: Finalized invoices remain editable in this system — the status is informational only.
Auto-persistence of WMS data: After saving, when you reopen the invoice, all WMS source data (order rows in the Inbound/B2B tables) is restored automatically. You do NOT need to re-fetch from WMS. Click Load WMS Data only when you want to refresh/overwrite with new data from WMS.

14Excel Export

Click Export Combined (or the download icon on the invoice list) to generate the Excel file. A preview dialog opens first.

Export Preview Dialog

  1. Review the Summary
    The dialog shows all billing categories, customer totals, OM Subsidy, and Office Sublease with the same layout as the Summary panel.
  2. Configure options
    Set Storage snapshot dates and other parameters if needed.
  3. Click Download Excel
    The .xlsx file is generated client-side and downloaded immediately.

Excel Workbook Structure

SheetContents
SummaryAggregated view across all customers. All quantities and amounts are formula-linked to customer sheets (no hardcoded values).
[CUSTOMER]_InvoiceStyled invoice for each customer — one per tab. Contains all 6 billing categories with quantities, rates, and amounts.
[CUSTOMER]_Inbound15-column raw inbound order detail table with SUM formulas linking to the customer invoice sheet.
[CUSTOMER]_B2B15-column B2B shipping order detail table with SUM formulas.
[CUST]_Storage_AvgPer-customer storage type averages — links to invoice storage quantities.
Storage_YYYY-MM-DDRaw inventory snapshot for the 15th and last day of the month.
Rate TableStandard rate schedule for reference.
OM SubsidyOM Subsidy calculation worksheet with all inputs and the formula breakdown.
Per Customer BreakdownIn the Summary sheet: a section showing each customer's subtotal by billing category (cross-sheet formulas).
All values in the Excel are formula-driven. The Summary and Per Customer Breakdown sections reference the customer sheets with =SUM() and =SUMIF() formulas — you can modify quantities in any customer sheet and the totals update automatically.

How to Set Custom Rates

  1. Select the customer from the dropdown. If they have no saved rates, all fields show the default rates.
  2. Enter custom rates in the input fields next to each billing item. Leave blank (or enter the default value) to use the standard rate.
  3. Click Save Rate Master. Rates are stored in Redis keyed by customer code.
  4. Create a new invoice for this customer — custom rates are automatically applied. The rate version in the header will show custom (date).
Existing invoices are NOT affected when you update the Rate Master. Only newly created invoices pick up the saved rates. To apply new rates to an existing invoice, delete and recreate it.
