WMS Dashboard
User Guide & SOP

Complete reference for all modules: Receiving, Shipping, Inventory, Locations, Products, Returns, and Billing. Follow this guide to operate the warehouse management system efficiently.

📦 Receiving & Stow 🚚 Shipping (B2B/B2C/B2S/B2E) 📊 Inventory & Snapshots 🧾 Billing & Invoicing

📋 Overview

Spider WMS Dashboard is a web-based Warehouse Management System for STL Logistics. It connects directly to the WMS API and Supabase database to give you real-time visibility into all warehouse operations.

📊
Real-Time Dashboard
Live KPIs for receiving, shipping, inventory, and locations. Auto-refreshes every 5 minutes.
📦
Receiving & Stow
Process inbound orders step by step with barcode scanning and stow tag workflow.
🚚
Shipping (4 Types)
B2B, B2C, B2S, B2E shipping management with status tracking and packing checklist.
📍
Location Master
View, search, and bulk-upload warehouse locations with zone/aisle/bay/level/position.
🕒
Inventory Snapshots
Daily snapshots stored in Supabase. Browse inventory as of any past date.
🧾
Billing & Invoicing
Auto-calculate customer invoices from WMS data. Export to Excel. Manage rate masters.

System Requirements

  • Modern browser: Chrome 100+, Edge 100+, Safari 16+
  • Screen resolution: 1280×768 minimum (1440×900 recommended)
  • Internet connection required (calls WMS API in real time)
  • Valid WMS user account with appropriate permissions
🔐

Login

Authenticate with your WMS credentials. Your session is stored in browser memory and cleared on logout.

How to Log In

  1. Open the app URL in your browser. You will be redirected to /login automatically if not authenticated.
  2. Enter your User ID and Password (same credentials as the WMS system).
  3. Click Sign In. The system authenticates against the WMS API and stores your bearer token.
  4. On success, you are redirected to the Dashboard.

A circular countdown ring (top right) shows time until the next auto-refresh (every 5 minutes). When below 30 seconds it turns red with a pulse. Click Refresh anytime to force an immediate reload.

Toggle between By Type (animated donut chart) and By Zone (bar chart). Hover donut segments to see counts and percentages per location type.

Receiving Pipeline

Four status buckets shown as clickable cards. Click any card to filter the recent orders table to that status. Click again to clear.

  • AA · Pre-Alert — order registered, shipment not yet arrived
  • CA · Processing — actively being received
  • DA · Complete — receiving finished
  • EA · Hold — receiving on hold, issue found

Viewing Orders

  1. Go to Receiving → Receiving Orders.
  2. Orders load automatically. Search box filters by order code, customer, or any field.
  3. Filter dropdowns narrow by Status, Warehouse, and Customer.
  4. Pagination at the bottom — change page size to 10 / 30 / 50 / 100.
  5. Click any row to open the detail panel on the right.

Detail Panel Tabs

  • Info — order header (customer, dates, warehouse, container info)
  • Items — line items: SKU, quantity, condition per item
  • Docs — attached shipping documents
  • Recv Info — internal notes (carton count, pallet count, container size, notes — saved to Supabase)
  • Raw — full API JSON response for debugging

Changing Order Status

Status changes are sent directly to the WMS API. Only advance status forward — backward transitions are blocked (except EA Hold which is always allowed).
  1. Click an order to open the detail panel.
  2. Click Change Status in the Info tab.
  3. Select the new status. For EA (Hold), a comment box appears — enter the reason.
  4. Click Confirm. The list updates immediately.

Receiving Process

Step-by-step barcode-scan workflow for physically receiving inventory at the dock. Best used on a tablet.

  1. Go to Receiving → Receiving Process.
  2. Open orders (AA = Pre-Alert, CA = In-Progress) are listed with stow tag counts.
  3. Scan the order barcode in the input box, or click Start next to an order.
  4. The order transitions to CA (Processing) if it was AA.
  5. You are taken to the item-by-item scan screen for that order.
  6. Scan each item barcode and enter received quantities. Items turn green when confirmed.
  7. When all items are done, click Complete Receiving to finalize (status → DA).
Stow tags are automatically generated for received items and appear in the Stow Process queue immediately.

Shipping

Four shipping types sharing the same interface: B2B, B2C, B2S, and B2E.

🏢
B2B — Business to Business
Bulk orders to partners or distributors. Typically pallet-level shipments.
👤
B2C — Business to Consumer
Direct-to-consumer e-commerce. Individual package shipments.
🏪
B2S — Business to Store
Replenishment orders to retail store locations.
🌐
B2E — Business to eCommerce
Marketplace fulfillment (Amazon, Walmart, etc.).

Order List

  1. Navigate to Shipping → B2B (or B2C / B2S / B2E).
  2. Orders load automatically. Search by order code, customer, tracking number, or address.
  3. Filter by status using the dropdown.
  4. Click Export to download visible rows as Excel.
  5. Click any row to open the detail panel.

Detail Panel Tabs

  • Info — receiver, address, carrier, tracking number, dimensions, weight
  • Items — SKU-level line items with locations and quantities
  • Docs — attached shipping documents
  • Raw — full JSON for debugging

Changing Shipping Status

  1. Open the order detail panel and click Change Status.
  2. Select the new status. Only forward transitions are shown (HA and CC are always available).
  3. Click Confirm. The change is pushed to the WMS API immediately.
CC (Cancellation) requires a comment. Once cancelled, an order cannot be reactivated.

Packing

Guided item-scan checklist for warehouse staff packing outbound orders.

  1. Go to Shipping → Packing.
  2. Orders with status CA (Packing Request) are listed.
  3. Scan the order barcode or click the order to start.
  4. The packing screen shows each item: SKU, location, lot, required quantity.
  5. Scan each item barcode. The circle turns to a green ✓ when scanned.
  6. Ship-from and ship-to address cards are displayed for label printing reference.
  7. When all items are checked, click Complete — the order advances to DA (Packing Complete).
Packing progress is saved to Supabase automatically. You can close the browser and resume the same packing session later.

Returns

View customer return orders from the WMS return order API.

  1. Go to Returns in the left sidebar.
  2. Return orders load automatically (up to 100 records).
  3. Click Refresh to reload the latest data.
  4. The table columns are auto-detected from the API response.
Returns currently displays WMS data read-only. To process or update a return, use the WMS system directly.

Inventory Inquiry

Live inventory search across all warehouses. Supports single adjustments and bulk CSV upload.

Searching Inventory

  1. Go to Inventory → Inventory Inquiry.
  2. Select a Warehouse (required). Optionally filter by Customer.
  3. Use the search box to filter by SKU, product name, or location code.
  4. Results show: Location, SKU, Product Name, Qty, Available Qty, Lot, Expire Date, Condition, Zone.
  5. Click Export to download visible rows as Excel.

Inventory Adjustment (Single)

  1. Click + Adjust (blue button, top right).
  2. Select warehouse and customer.
  3. Enter or scan the location code. Auto-fills zone/aisle/bay/level/position.
  4. Enter SKU, condition, quantity, lot, expire date, and remark.
  5. Click Submit Adjustment. The WMS API processes the change immediately.

Batch Adjustment (CSV Upload)

  1. Click the Upload button to open the batch panel.
  2. Prepare CSV with headers: locationCode, sku, condition, adjustQty, lotNo, expireDate, remark
  3. Drag & drop or select your CSV file. Preview rows appear with detected columns.
  4. Click Submit All. Each row is processed with a progress bar.
  5. Results show ✓ success or ✗ error per row with messages.
Batch adjustments cannot be undone. Negative quantities reduce stock. Always verify your CSV before submitting.

Inventory History

Daily inventory snapshots stored in Supabase. Browse inventory as it was on any past date.

Viewing a Snapshot

  1. Go to Inventory → History.
  2. Select a date (defaults to today) and Warehouse.
  3. Click Load. Snapshot data loads from Supabase.
  4. Table shows: Location, SKU, Product Name, Qty, Available Qty, Lot, Expire Date, Captured At (LA timezone).
  5. Use the search box to filter. Click Export CSV to download.
Snapshots are captured automatically by a nightly cron job. If today's snapshot is missing, click Save to DB on the History page to trigger a manual capture.

Using History for Storage Billing

The Billing module reads History snapshots to compute storage fees. It fetches the 15th-of-month and last-day-of-month snapshots and averages them to determine billed storage quantities. See Storage Billing for details.

Location Conflicts

Finds locations where multiple customers have inventory — a compliance and contamination risk.

  1. Go to Inventory → Location Conflicts.
  2. The system scans all inventory and identifies locations shared by 2+ customers.
  3. Conflicted locations are listed with each customer's items and quantities.
  4. Direct staff to move inventory to dedicated single-customer locations to resolve conflicts.
Resolve location conflicts promptly to prevent cross-customer contamination and inaccurate inventory counts.

Location Master

View all warehouse locations. Search, filter, and bulk-upload new locations.

Viewing Locations

  1. Go to Locations → Location Master.
  2. Select a Warehouse. Locations load automatically (up to 9,999 records).
  3. Search box filters by location code, zone, or occupancy type.

Location Code Structure

Each location has a 5-part code: Zone – Aisle – Bay – Level – Position

Example: 01-31-11-04-01 = Zone 01, Aisle 31, Bay 11, Level 04, Position 01

Bulk Upload

  1. Click Upload to open the upload panel.
  2. Prepare a CSV/Excel file with headers:
    zoneNm, aisleNm, bayNm, levelNm, positionNm, maxCbm, maxCbf, occupancyInfo, remark
  3. Drag & drop or click to select the file. Preview rows appear.
  4. Click Upload All. Each location is created via the WMS API with progress shown.
  5. Results show ✓ / ✗ per row with error details.
The occupancyInfo field determines storage billing type. Use exact spelling: Bin, Shelf, Carton, Pallet Short, Pallet Regular, Pallet Tall, Open Floor.

Available Locations

Shows only empty locations — useful for assigning stow destinations.

  1. Go to Locations → Available Locations.
  2. Select a warehouse. The page cross-references inventory data to identify empty slots.
  3. Filter by zone or location type to find the right spot for incoming goods.

WMS Source Data (Auto-Fill)

Load raw WMS order data to auto-populate invoice quantities. Once loaded, data persists with the invoice.

Loading WMS Data

  1. Open an invoice in the editor.
  2. In the WMS Data panel, select the Warehouse and confirm the Billing Period.
  3. Click Load WMS Data. The system fetches receiving, B2B, B2C, and return orders for the period.
  4. Invoice quantities are auto-populated (highlighted in blue).
  5. Data is saved with the invoice — reopen the invoice anytime without re-fetching.
Load WMS Data is a reset button — clicking it again re-fetches from the API and overwrites manual edits. Use only to start fresh or fix a mistake.

WMS Source Sub-Tabs

  • Inbound — receiving orders with carton/pallet/SKU counts per order
  • B2B — B2B shipping orders with pick/carton/packing/labor per order
  • B2C — B2C shipping orders with pick counts and fragile items
  • Returns — return orders processed in the billing period

Each row in the source tables has editable fields. Editing any field automatically recalculates invoice line item totals.

Storage Billing

Storage fees use daily inventory snapshots. The system averages the 15th-of-month and last-day-of-month to determine billed quantities.

How Storage Is Calculated

  • Billed quantity = average of (snapshot on the 15th + snapshot on the last day)
  • Quantities count number of occupied locations by type — not units of inventory
  • Location types starting with "Re" (Re Bin, Re Shelf, etc.) count as their base type

Loading Storage Data

  1. In the invoice editor, switch to the Storage tab in the WMS Data panel.
  2. Click Load WMS History. Fetches snapshots for the 15th and last day of the billing period.
  3. Storage quantities auto-apply to the Storage line items in the invoice.
  4. Click Storage Summary (right panel tab) to see the per-customer, per-type grid.

Summary Tab

A full financial overview of the invoice, suitable for review before finalization.

Opening the Summary

Click Summary in the right-side tab bar of the invoice editor.

Contents

  • Per-category subtotals: Inbound, Storage, Fulfillment B2B, Fulfillment B2C, Returns, Labor
  • Grand Total — sum of all billing categories
  • OM Subsidy — shown separately below Grand Total
  • Office Sublease — shown separately below Grand Total
  • Multi-customer groups: each customer in a column + group total column
OM Subsidy and Office Sublease are excluded from Grand Total because they are billed separately with different payment terms.

💡 Tips & Shortcuts

General Navigation

  • Click the STL WMS logo (top of sidebar) to return to Dashboard from anywhere.
  • Sidebar groups collapse/expand by clicking the group header (Inventory, Shipping, Receiving, etc.).
  • The active page is highlighted in blue.
  • Browser Back/Forward work — the app is fully router-based (Next.js App Router).

Dashboard

  • KPI cards are clickable — they navigate to the related page.
  • Click a Receiving Pipeline status card to filter the order table below.
  • The auto-refresh timer resets whenever you click Refresh manually.
  • Hover donut chart segments to see detailed counts.

Receiving & Stow

  • The scan input box auto-focuses on page load — just start scanning.
  • Receiving Info saves automatically to Supabase — no need to re-enter on revisit.
  • Status changes are logged in the WMS and cannot be undone (except EA ↔ any status).
  • Stow tags persist even if you close the browser. Resume them from the pending list.

Shipping

  • All 4 shipping types (B2B/B2C/B2S/B2E) share the same interface. The color accent differs.
  • The Export button downloads all currently visible rows — use status filter to export subsets.
  • Packing sessions save automatically — staff can resume an interrupted packing job.

Billing

  • Always Load WMS Data first before editing quantities — it sets the auto-fill baseline.
  • Blue-highlighted Qty fields were auto-filled — verify them before finalizing.
  • WMS source data is saved with the invoice. No need to re-load on subsequent opens.
  • Set Rate Master rates for a customer before creating their invoice for automatic rate application.
  • For multi-customer billing periods, create a group invoice (select multiple customers) to export together.
  • The Storage Summary panel appears only after loading WMS History data.
  • OM Subsidy and Office Sublease totals are always shown separately from the Grand Total.

Inventory & Locations

  • If today's inventory snapshot is missing in History, click Save to DB to trigger a manual capture.
  • Check Location Conflicts weekly to catch cross-customer storage issues early.
  • Batch CSV upload is the fastest way to adjust large quantities — prepare the CSV in Excel first.
  • The occupancyInfo field in location uploads must match exact spelling for storage billing to work.
When in doubt, click the Refresh button on any page to re-fetch the latest WMS data. All pages show the timestamp of the last successful load.
